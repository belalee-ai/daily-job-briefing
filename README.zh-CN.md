# 每日职位更新简报

> [English](README.md)

一个 Claude Code Skill，帮你每天自动搜索目标城市、目标行业的最新招聘信息，输出一份包含岗位推荐、薪资分析、市场趋势和行动清单的求职日报。

## 为什么做这个 Skill？

求职者每天花大量时间在多个平台之间来回翻，脑子里过滤不相关的 JD，看过的岗位转头就忘。这个 skill 的思路是做你的**求职情报官**：

1. **你只需要描述自己一次** — 目标城市、行业、岗位、薪资区间、偏好 — 形成持久化的求职画像，驱动后续所有搜索
2. **它替你搜** — 跨公司官网、招聘平台、LinkedIn、行业媒体，用三层可信度体系标注信源质量，让你知道哪些信息靠谱
3. **它替你想** — 每个岗位按 4 个加权维度对照**你的画像**打分，不是泛泛的"挺合适"
4. **它告诉你该做什么** — 每份简报以 P0/P1/P2 优先级的行动清单收尾，不只是丢一堆链接

目标：打开简报，花 1 分钟，就知道今天有什么值得投的。

## 功能亮点

- **个性化匹配** — 首次使用时 8 个问题快速建档（约 3 分钟），后续所有推荐基于你的画像定制
- **三层信源体系** — 公司官方招聘页（高可信）→ 招聘平台 + LinkedIn（中等）→ 行业媒体和社区（参考）。每个岗位标注信源层级，让你知道置信度
- **四维评分**（满分 5.0）— 经验匹配(35%) + 薪资竞争力(25%) + 成长潜力(20%) + 求职因素匹配(20%)。按得分分梯队：第一梯队（≥4.0，最多 5 个）→ 第二梯队（3.0–3.9）→ 第三梯队（2.5–2.9）
- **6 种模式** — 完整日报 / 快速日报 / 定向追踪 / 历史回看 / 标记追踪 / 市场概览
- **标记系统** — 对话中说"标记XX"即可持续关注某岗位，14 天无进展自动衰减和归档
- **多行业覆盖** — 预置 AI、互联网、金融科技、游戏、新能源、电商 6 大行业的公司清单和搜索模板
- **历史感知** — 自动比对最近 3 天日报，去重、追踪状态变化、标注真正的新发现（🆕）

## 工作流程

```
首次建档（一次性）            每日简报
┌─────────────┐     ┌──────────────────────────────────────────┐
│ 8 个问题     │     │ Step 1：多源岗位采集                       │
│ → 生成画像   │────▶│ Step 2：市场行情扫描                       │
│              │     │ Step 3：四维匹配评估                       │
│              │     │ Step 4：生成结构化日报                     │
│              │     │ Step 5：保存与索引                         │
└─────────────┘     └──────────────────────────────────────────┘
```

**Step 1 — 岗位采集**：从三层信源搜索（官网 → 平台 → 媒体），总量上限约 15 次。每条岗位记录：公司、岗位、方向/BU、城市、薪资、状态、来源链接、信源层级、时间。

**Step 2 — 行情扫描**：扫描 4 个维度（薪资行情、供需关系、热门方向、行业事件），捕捉影响你求职策略的信号。

**Step 3 — 匹配评估**：每个岗位按四维加权模型对照你的画像打分。匹配分析会引用你的**具体经验**——不是模糊的"你很合适"。

**Step 4 — 日报输出**：Top 3 岗位做深度分析：为什么匹配（引用你的经验）、面试话术建议（50–80 字）、简历配置建议、具体投递渠道推荐。

**Step 5 — 保存导航**：日报存入 `shared/reports/`，自动维护索引文件方便回看历史。

## 使用方式

安装后在 Claude 中使用以下触发词：

| 说法 | 效果 |
|------|------|
| "职位日报" / "今天有什么新岗位" | 生成完整日报 |
| "快速日报" / "简版" | 只看第一梯队的精简版 |
| "XX公司有更新吗" | 定向追踪某家公司 |
| "我标记的" / "追踪进展" | 查看所有标记岗位的最新动态 |
| "市场行情" / "薪资趋势" | 专项市场分析 |
| "更新我的信息" | 修改目标城市/行业/薪资等偏好 |

首次使用时会引导你完成 8 个问题的快速设置（约 3 分钟）。

## 质量标准

每份日报自动执行以下规则：

- **信息完整性** — 每条岗位必须包含 7 要素：公司、岗位、方向、城市、薪资、状态、来源链接。缺项标"待确认"
- **薪资可信度** — 标注 ✅ 官方发布 / ⚠️ 综合预估 / ❓ 未验证
- **禁止编造链接** — 来源 URL 必须来自实际搜索结果；无直链时标注平台名
- **诚实匹配** — 匹配分析必须引用画像中的具体经验，差距如实说明，不夸大
- **梯队纪律** — 第一梯队最多 5 个岗位，与第二梯队至少 1.0 分差距
- **风险透明** — 公司负面信息（裁员、亏损等）必须体现，标注来源

## 可选增强

- **简历匹配** — 将简历放入 `shared/resume/` 目录，日报会引用你的具体项目数据做更深度的匹配分析
- **Web Access Skill** — 安装 [web-access](https://github.com/eze-is/web-access) 可解锁浏览器抓取、登录态访问等增强搜索能力

## 数据与隐私

本 skill 在运行时会在 `shared/` 目录下创建以下本地文件：

- `user-profile.md` — 你的求职画像（目标城市、行业、岗位、薪资期望等）
- `bookmarks.md` — 你标记关注的岗位清单
- `resume/` — 你自行放置的简历文件
- `reports/` — 生成的日报文件

**所有数据仅存储在你的本机**，不会上传到任何服务器。上述文件已在 `.gitignore` 中排除，不会被 git 提交。如果你 fork 或分享本仓库，请确认 `shared/` 目录下无个人数据。
